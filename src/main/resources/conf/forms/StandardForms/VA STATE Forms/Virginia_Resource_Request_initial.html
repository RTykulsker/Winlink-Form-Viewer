Resource Request

Introduction.

This Resource Request is designed for delivery by radio when WebEOC is unavailable. It is based upon the paper form to the extent possible within the constraints of Winlink Express.

* NOTE on printing:

Printing this form is dependent upon your selected printer and its settings. Various Web Browsers and and their viewing sizes are not all the same. Printing from a browser's print function may be different from the operating system's default print mode. Select what works for you, for both printing and viewing. You may print direct to the printer as an HTML or save as a PDF, then print the PDF. In some instances Landscape mode may print better than Portrait.

NOTE: The Save/Load feature will enter the previous "saved" data. So if dates/times, or any other information needs to be CHANGED, do so prior to submitting.

Purpose.

The Resource Request form is used to request resources that cannot be supplied from within the requesting agency or jurisdiction. When delivered, the complete message information is dispayed in plain text within the message body so that non-Express users can read, since they do not have the rendered HTML.

A copy of the Resource Request should be sent to and maintained within the Documentation Unit. A copy of all your Express traffic is in your Sent Items folder. You may wish to create a Personal Folder for the event and move your desired traffic into it. Winlink Express has an Export/Import feature so you can save this folder on your desktop or elsewhere if desired. You can even send the folder to another Express user for them to import.
